# Export guide

`export.js`は、ホームディレクトリにあるファイルを、公開用ファイルとして同じ相対パスでこのリポジトリへ同期するためのスクリプトです。Node.js 18以降で動作し、外部パッケージには依存しません。

## コピー対象を設定する

コピー対象は`export.yaml`の`paths`に記述します。各パスは、ホームディレクトリとリポジトリルートの両方を基準にした相対パスです。

たとえば、次の指定では`~/.codex/AGENTS.md`をリポジトリの`.codex/AGENTS.md`へコピーします。

```yaml
paths:
  - .codex/AGENTS.md
```

ファイルとディレクトリのどちらも指定できます。ディレクトリを指定した場合は、コピー元からなくなったコピー先のファイルも削除して、ディレクトリ内部を同期します。ファイルを指定した場合は、そのファイルだけを追加または更新します。`export.yaml`には、公開用として管理する対象だけを追加してください。重複するパスや、一方が他方を含むパスは指定できません。

## ファイルの一部だけを書き出す

テキストファイルに次のマーカーが対で含まれている場合は、その区間と隣接するコメントブロックだけを書き出します。

```text
# Shared settings start here.
...
# Shared settings end here.
```

現在の`.codex/config.toml`は、この仕組みでローカル専用設定を除外しています。開始・終了マーカーが対応していない場合や、マーカーが不正に入れ子になっている場合はエラーになります。

## 書き出し前の検査

コピー前には、次のような個人情報・機密情報の候補を検査します。

- 機密性の高いファイル名
- ローカルのユーザー名とホームディレクトリのパス
- 一般的なAPIキー、アクセストークン、秘密鍵、パスワードのパターン

正確に`# ENV:`または`# NOTE:`で始まるコメント行では、ローカルのユーザー名を意図的な記述として許可します。その行でも、ユーザー名以外の検査は継続します。

バイナリファイルはファイル名だけを検査し、内容は検査しません。また、この検査は誤公開を完全に防ぐものではありません。書き出したファイルは必ず人が確認してください。

## 使い方

リポジトリのルートで、最初にドライランを実行します。コピー予定、削除予定、検査結果が表示され、ファイルは変更されません。

```powershell
npm run check
```

問題がなければ、ファイルを書き出します。`npm run build`でも変更前に同じ検査を実行し、問題が見つかった場合はコピーも削除も行いません。削除対象は、`export.yaml`で指定されたコピー先ディレクトリの内部に限定されます。コピー元のファイルは変更しません。

```powershell
npm run build
```

最後に、追加・更新されたファイルを確認します。

```powershell
git status --short
git diff
```

`git diff`に表示されるのは追跡済みファイルの差分だけです。未追跡ファイルは個別に開いて内容を確認してください。

引数を付けずに`node export.js`を実行するか、`--help`を指定すると簡易ヘルプを表示します。

```powershell
node export.js --help
```

`export.yaml`からディレクトリ自体を外した場合、そのディレクトリは同期対象ではなくなるため、以前書き出したファイルは自動削除されません。必要に応じて手動で整理してください。コピー先にシンボリックリンクが含まれている場合は、安全のため処理を中止します。
